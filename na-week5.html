직교 행렬 (Orthogonal Matrix)

📖 정의

행렬 A의 전치(Aᵀ)가 A의 역행렬과 같으면 A를 직교 행렬(Orthogonal Matrix)이라고 합니다.

정의: A · Aᵀ = I ←→ Aᵀ = A⁻¹ (즉: 전치행렬 = 역행렬)
💡 직관: 직교 행렬은 "단위 기저 벡터들이 서로 수직이고 길이가 1인" 특수한 행렬입니다.
🎯 핵심 장점: 역행렬 계산이 매우 쉬움! M⁻¹ = Mᵀ (전치만 하면 됨)

정규직교 (Orthonormal)

두 벡터가 정규직교(Orthonormal)하다는 것은:

Orthogonal (직교)
두 벡터가 수직 → 내적 = 0
V₁ · V₂ = 0
Normalized (단위화)
벡터의 길이 = 1
|V| = 1
정규직교 조건: Vᵢ · Vⱼ = δᵢⱼ (Kronecker delta) δᵢⱼ = { 1 (i = j) { 0 (i ≠ j)

🔢 직교 행렬 예시 (2D 회전 행렬)

R = [cosθ -sinθ] [sinθ cosθ] Rᵀ = [cosθ sinθ] [-sinθ cosθ] R · Rᵀ = [cosθ -sinθ] [cosθ sinθ] [sinθ cosθ] [-sinθ cosθ] = [cos²θ + sin²θ 0 ] [0 sin²θ + cos²θ] = [1 0] = I ✅ [0 1]
sin²θ + cos²θ = 1 이기 때문에 항등행렬이 됩니다!

정리: 직교 행렬이면 길이와 각도를 보존

직교 행렬 M은 변환 후에도 길이와 각도를 바꾸지 않습니다.

① 길이 보존 증명

벡터 X의 길이: |X| = √(X · X) = √(XᵀX) MX의 길이: |MX|² = (MX)ᵀ(MX) = XᵀMᵀMX = XᵀIX = XᵀX = |X|² ∴ |MX| = |X| ← 길이 보존! ✅

② 각도 보존 증명

X, Y 사이의 각도 θ: cosθ = X·Y / (|X||Y|) MX, MY 사이의 내적: (MX) · (MY) = (MX)ᵀ(MY) = XᵀMᵀMY = XᵀIY = X·Y → 내적 값이 그대로 → 각도 보존! ✅
💡 회전 행렬이 직교 행렬이기 때문에 회전이 거리와 각도를 보존하는 것입니다!

3D 회전 행렬들의 관계

Rx, Ry, Rz: 각 축에 대한 회전 행렬 (RxRyRz)⁻¹ = Rz⁻¹Ry⁻¹Rx⁻¹ = RzᵀRyᵀRxᵀ = (RxRyRz)ᵀ → 연속 회전의 역행렬 = 전치행렬!

📊 직교 행렬 핵심 정리

성질내용
정의AAᵀ = I (즉, Aᵀ = A⁻¹)
역행렬Aᵀ만 계산하면 됨 (매우 간단!)
길이 보존|MX| = |X| 항상 성립
각도 보존MX·MY = X·Y 항상 성립
행렬식det(A) = ±1

법선 벡터 변환 (Transforming Normal Vectors)

문제: 법선 벡터를 그냥 변환하면 안 된다?

표면을 변환할 때 법선 벡터(Normal Vector)도 함께 변환해야 합니다. 그런데 표면과 같은 행렬로 변환하면 법선이 틀어질 수 있습니다!

⚠️ Non-uniform Scaling처럼 축마다 다른 스케일이 적용되면 법선 벡터가 표면에 더 이상 수직이 아니게 됨!

올바른 법선 변환 행렬 G 유도

표면 위의 접선 벡터(Tangent Vector) T와 법선 N은 항상 수직: N · T = 0

변환 후에도 수직이어야 하므로: GN · MT = 0

1
수직 조건: (GN)ᵀ(MT) = 0
2
NᵀGᵀMT = 0
3
Nᵀ(GᵀM)T = 0 이어야 하는데, N·T=0이므로 GᵀM = I 이어야 함
4
∴ Gᵀ = M⁻¹ → G = (M⁻¹)ᵀ = (Mᵀ)⁻¹
법선 벡터 변환 행렬: G = (M⁻¹)ᵀ = (Mᵀ)⁻¹ 법선 벡터 변환: N' = G·N = (M⁻¹)ᵀ · N
🎯 M이 직교 행렬이면? M⁻¹ = Mᵀ 이므로 G = (Mᵀ)ᵀ = M → 그냥 M을 써도 됨!
즉, 회전 변환에서는 법선도 그냥 동일한 R로 변환하면 됩니다.

🔢 예시: Non-uniform Scaling에서 법선 변환

x방향 2배, y방향 1배 스케일 변환: M = [2 0] [0 1] 올바른 법선 변환 행렬: M⁻¹ = [1/2 0] [0 1] G = (M⁻¹)ᵀ = [1/2 0]ᵀ = [1/2 0] [0 1] [0 1] 원래 법선 N = [0, 1]ᵀ (위쪽 방향) G·N = [1/2 0] [0] [0] [0 1] [1] = [1] → 정규화하면 [0, 1] (올바름!) 만약 그냥 M을 쓰면: M·N = [2 0] [0] [0] [0 1] [1] = [1] → 이 경우는 우연히 같지만, 일반적으론 틀림

2D 좌표계 변환 (Translation of Coordinate System)

핵심 아이디어: 좌표계 이동 = 점의 반대 이동

좌표계를 이동하는 것과 좌표계는 고정하고 점을 반대 방향으로 이동하는 것은 수학적으로 동일합니다!

좌표계를 T만큼 이동 → 점의 새 좌표: X' = T⁻¹ · X = X - T (이동 행렬의 역: 반대 방향 이동)

좌표계 회전

좌표계를 θ만큼 회전 = 점을 -θ만큼 회전한 것과 동일

좌표계를 R만큼 회전 → 점의 새 좌표: X' = R⁻¹ · X = Rᵀ · X (직교 행렬이므로 R⁻¹ = Rᵀ)

회전 + 이동 동시 적용

좌표계를 먼저 이동(T)하고 나서 회전(R)하는 경우:

X' = (TR)⁻¹ · X = R⁻¹ · T⁻¹ · X (역행렬은 순서 반대!)
💡 TR은 "T 이동 먼저, R 회전 다음". 역변환은 R⁻¹ 먼저, T⁻¹ 다음.

3D 좌표계 변환 (Transformation of 3D Coordinate System)

새 좌표계의 정의

3D에서 새로운 좌표계는 다음 3가지로 완전히 정의됩니다:

① Origin (원점)
X₀ = (x₀, y₀, z₀)
② Coordinate Axes (축 방향)
U = (uₓ, u_y, u_z) ← 새 x축
V = (vₓ, v_y, v_z) ← 새 y축
N = (nₓ, n_y, n_z) ← 새 z축
💡 N은 U, V에서 외적으로 구할 수 있음:
N = (U × V) / |U × V|

3D 좌표계 변환: 2단계

세계 좌표계(World)에서 새 좌표계(Local)로 변환하는 과정:

1
원점 이동 (Translation)
새 좌표계의 원점 X₀를 세계 원점으로 이동
이동 행렬 T⁻¹: [1 0 0 -x₀] [0 1 0 -y₀] [0 0 1 -z₀] [0 0 0 1]
2
축 회전 (Rotation)
원래 축 (x,y,z)를 새 축 (U,V,N)으로 정렬
회전 행렬 R⁻¹: 행: U, V, N 벡터를 행(row)에 배치 [uₓ u_y u_z 0] [vₓ v_y v_z 0] [nₓ n_y n_z 0] [0 0 0 1] (이 행렬의 역행렬 = 전치행렬, ∵ 직교 행렬)
전체 변환: X' = R⁻¹ · T⁻¹ · X 풀어서: X' = [U 0] [I -X₀] X [V 0] [0 1 ] [N 0] [0 1]
💡 왜 행(row)에 U, V, N을 넣나? 행렬 곱에서 회전 행렬의 행이 새 좌표계 축 방향과의 내적을 계산하기 때문. X를 U 방향으로 투영 = U·X = 새 x좌표!

🔢 완전 예시: 2D 좌표계 변환

새 원점 T=(3,0), 새 x축이 45° 방향일 때, 점 X=(5,3)의 새 좌표는?

1
이동: X - T = (5-3, 3-0) = (2, 3)
2
새 축: U = (cos45°, sin45°) = (√2/2, √2/2)
V = (-sin45°, cos45°) = (-√2/2, √2/2)
3
내적으로 새 좌표 계산:
x' = U·(2,3) = (√2/2)·2 + (√2/2)·3 = 5√2/2 ≈ 3.54
y' = V·(2,3) = (-√2/2)·2 + (√2/2)·3 = √2/2 ≈ 0.71
결과: X' ≈ (3.54, 0.71)

📊 좌표계 변환 핵심 요약

변환수식직관
이동만X' = T⁻¹X좌표계 이동 = 점 반대 이동
회전만X' = R⁻¹X = RᵀX좌표계 회전 = 점 역방향 회전
이동+회전X' = R⁻¹T⁻¹X역순으로 역변환 적용
3D 일반X' = Rᵀ(X - X₀)원점이동 후 축정렬
⚠️ 순서 주의: T먼저 적용하고 R적용 → 역은 R⁻¹ 먼저, T⁻¹ 나중!
