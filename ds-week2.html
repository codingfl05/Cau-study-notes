Algorithm — 정의 & 5가지 요건

알고리즘이란 특정 작업을 수행하는 유한한(finite) 명령어 집합입니다. 레시피와 같아요. 요리를 만드는 순서가 명확하고, 언젠가는 끝나야 하죠.

요건의미예시
Input출력을 만들기 위해 사용되는 입력 데이터숫자 배열 [5, 3, 8, 6, 2]
Output알고리즘이 생성하는 결과정렬된 배열 [2, 3, 5, 6, 8]
Definiteness각 명령이 명확하고 모호하지 않아야 함"적당히 더해" ← ❌ 모호함
Finiteness유한한 단계 후 종료되어야 함무한루프는 알고리즘이 아님
Effectiveness불필요하고 중복된 연산이 없어야 함x + 0을 반복하는 건 낭비
⚠️ 무한루프(infinite loop)가 있으면 Finiteness를 위반 — 알고리즘이 아닙니다!

Sorting — 왜 배울까?

정렬(Sorting)은 데이터를 특정 순서(오름차순 또는 내림차순)로 배치하는 과정입니다. 전화번호부, 검색 결과, 성적표… 어디서나 쓰이죠. 이 강의에서는 4가지 대표 정렬 알고리즘을 배웁니다.

💡 정렬 알고리즘은 "어떻게 하면 더 빠르게 정렬할 수 있을까?"라는 Algorithm 설계의 교과서적 예제입니다.

Sorting Algorithms — 4가지 정렬

① Selection Sort (선택 정렬)

핵심 아이디어: "전체에서 가장 작은 것을 찾아서 맨 앞에 갖다 놓자!" 를 반복합니다.

마치 카드를 손에서 꺼낼 때, 가장 낮은 숫자를 골라 왼쪽에 쌓는 것과 같습니다.

🔍 동작 원리: 전체를 스캔 → 최솟값 찾기 → 맨 앞 원소와 교환 → 범위 줄이기 반복

void selectionSort(int arr[], int n) { int min_idx, temp; for (int i = 0; i < n - 1; i++) { min_idx = i; // 현재 위치를 최솟값으로 가정 for (int j = i + 1; j < n; j++) { if (arr[j] < arr[min_idx]) min_idx = j; // 더 작은 값 발견 시 갱신 } temp = arr[min_idx]; // swap arr[min_idx] = arr[i]; arr[i] = temp; } }
📊 시간 복잡도: O(n²) — 항상 이중 루프. n=5이면 약 10번 비교.

② Bubble Sort (버블 정렬)

핵심 아이디어: 인접한 두 원소를 비교해서 순서가 틀리면 교환하자! 큰 값이 거품처럼 오른쪽으로 떠오릅니다.

🫧 동작 원리: 왼쪽부터 이웃한 쌍을 비교 → 역순이면 swap → 한 Pass 끝나면 가장 큰 값이 맨 오른쪽에 정착

void bubbleSort(int arr[], int n) { int temp, swapped; for (int i = 0; i < n - 1; i++) { swapped = 0; for (int j = 0; j < n - i - 1; j++) { if (arr[j] > arr[j + 1]) { temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp; swapped = 1; } } if (!swapped) break; // 이미 정렬됐으면 조기 종료! } }
💡 swapped 플래그 덕분에 이미 정렬된 경우 O(n)으로 빠르게 종료. 최악은 O(n²).

🃏 동작 원리: 현재 원소(key)를 왼쪽 정렬된 구간에서 맞는 자리를 찾아 삽입

void insertionSort(int arr[], int n) { int key, j; for (int i = 1; i < n; i++) { key = arr[i]; // 삽입할 원소 j = i - 1; while (j >= 0 && arr[j] > key) { arr[j + 1] = arr[j]; // 한 칸씩 오른쪽으로 밀기 j--; } arr[j + 1] = key; // 빈 자리에 key 삽입 } }
💡 거의 정렬된 데이터에서 가장 빠름. 실전에서 많이 쓰이는 이유!

④ Merge Sort (병합 정렬)

핵심 아이디어: 분할 정복(Divide & Conquer)! 배열을 반으로 계속 쪼개다가, 다시 합치면서 정렬합니다.

비유: 두 팀으로 나눠서 각자 정렬한 다음, 합칠 때 순서대로 합치면 전체가 정렬됩니다.

✂️ 동작 원리: Divide(반으로 쪼개기) → Conquer(각각 재귀 정렬) → Merge(두 정렬된 배열 합치기)
if (left < right) { int mid = left + (right - left) / 2; mergeSort(arr, left, mid); // 왼쪽 반 정렬 mergeSort(arr, mid + 1, right); // 오른쪽 반 정렬 merge(arr, left, mid, right); // 합치기 }
📊 시간 복잡도: O(n log n) — Selection/Bubble/Insertion Sort의 O(n²)보다 훨씬 빠릅니다!

Bubble 인접 원소 비교·교환 O(n²) O(n) 이미 정렬 시 O(n)

⚠️ O(n²)과 O(n log n)의 차이: n=1,000,000이라면 → n²=1조 번 연산 vs n log n = 약 2천만 번 연산!

Recursive Algorithm (재귀 알고리즘)

재귀란 무엇인가?

재귀(Recursion)는 함수가 자기 자신을 호출하여 문제를 더 작은 부분 문제(subproblem)로 분해하는 기법입니다.

비유: 러시아 마트료시카 인형 — 안을 열면 또 같은 모양의 작은 인형이 나오고, 결국엔 더 이상 쪼갤 수 없는 가장 작은 인형이 나옵니다.

🔑 재귀의 필수 조건: (1) 자기 자신을 호출하는 재귀 케이스, (2) 호출을 멈추는 기저 케이스(base case)

예시 1: Fibonacci (피보나치)

F(0)=0, F(1)=1, F(n) = F(n-1) + F(n-2)

int fibonacci(int n) { if (n == 0) return 0; // base if (n == 1) return 1; // base return fibonacci(n-1) + fibonacci(n-2); // recursive }

F(5) = F(4)+F(3) = ... = 5

예시 2: Merge Sort

배열을 절반으로 쪼개 재귀적으로 정렬하고 합칩니다.

mergeSort(arr, left, right): mid = (left+right)/2 mergeSort(arr, left, mid) // ←재귀 mergeSort(arr, mid+1, right) // ←재귀 merge(arr, left, mid, right)

재귀가 곧 Merge Sort의 핵심!

재귀 알고리즘 장단점

✅ 장점

  • 알고리즘을 자연스럽게 표현 가능
  • 코드가 짧고 가독성이 높음
  • 작성·디버깅 시간이 줄어듦

❌ 단점

  • 함수 호출마다 스택 프레임 생성 → 메모리 더 사용
  • 반복문(iterative) 방식보다 느릴 수 있음
  • 스택 오버플로우(Stack Overflow) 위험
⚠️ 재귀 깊이가 너무 깊으면 스택 메모리가 가득 차 프로그램이 crash! 항상 base case를 먼저 설계하세요.

Abstract Data Type (ADT)

왜 ADT가 필요한가?

ADT란 데이터의 내부 구현 방식에 상관없이 동일한 인터페이스(interface)로 조작할 수 있게 하는 개념입니다.

비유: TV 리모컨처럼! 안에 회로가 어떻게 생겼든 버튼만 알면 채널을 바꿀 수 있죠. 사용자는 내부 구현을 몰라도 됩니다.

⭐ ADT는 "무엇(What)을 할 수 있나"를 정의하고, 구현은 "어떻게(How) 하나"를 결정합니다. 사용자는 What만 알면 됩니다.

Space Complexity (공간 복잡도)

알고리즘이 실행될 때 필요한 메모리 공간의 총량입니다.

Fixed Space (고정 공간)

  • 입력 크기와 무관한 상수 공간
  • 코드 자체, 상수값, static 변수
  • 예: int i; int temp; → 항상 8 bytes

Variable Space (가변 공간)

  • 입력 크기에 따라 변하는 공간
  • 재귀 스택 프레임, 동적 배열 등
  • 예: 재귀 n번 → n개의 스택 프레임

📝 예제: sum() vs recursive_sum()

int sum(int list[], int n) { int i; int temp_sum = 0; for (i=0; i<n; i++) temp_sum += list[i]; return temp_sum; } // Fixed: list*, n, i, temp_sum = 4×4 = 16 bytes // Variable: 없음 (0)
int recursive_sum(int list[], int n) { if (n) return recursive_sum(list, n-1) + list[n-1]; return 0; } // Fixed: list*, n = 2×4 = 8 bytes // Variable: list*, n, return_addr = 3×4 × n번 // → 재귀 깊이 n만큼 스택 쌓임!
⚠️ recursive_sum은 n이 커질수록 스택 메모리 n배 증가! 반복문 sum()이 공간 효율적입니다.

Time Complexity (시간 복잡도)

알고리즘이 입력 크기 n에 대해 실행하는 기본 연산(primitive operation)의 수입니다.

📝 예제: sum() vs recursive_sum() 시간 복잡도

sum()

  • 변수 선언: 2 steps
  • Loop 조건 체크: n+1 steps
  • 더하기: n steps
  • return: 1 step
  • = 2n + 4 steps → O(n)

recursive_sum()

  • if 체크: n+1 steps
  • 재귀 return: n steps
  • 최종 return: 1 step
  • = 2n + 2 steps → O(n)
💡 두 함수 모두 O(n)! 하지만 공간 복잡도는 다릅니다. 알고리즘 선택 시 시간과 공간을 모두 고려해야 합니다.

Asymptotic Analysis & Big-O 표기법

점근적 분석(Asymptotic Analysis)은 입력 크기 n이 충분히 클 때 알고리즘의 성능 경향을 수학적으로 표현합니다.

Big-O (O) — 상한(Upper Bound)
f(n) = O(g(n)) iff f(n) ≤ c·g(n), n > n₀
"f()는 g()보다 빠르게 증가하지 않는다"
Big-Ω (Omega) — 하한(Lower Bound)
f(n) = Ω(g(n)) iff f(n) ≥ c·g(n), n > n₀
"f()는 g()보다 느리게 증가하지 않는다"
Big-Θ (Theta) — 상한 + 하한 동시 (가장 정확)
f(n) = Θ(g(n)) iff c₁·g(n) ≤ f(n) ≤ c₂·g(n), n > n₀
"f()와 g()는 같은 성장 속도를 가진다" — Big-O보다 더 정확한 표현

Big-O 예제

예제 1: f(n) = 7n + 3 → O(n) c=8, n₀=3으로 설정하면: 7n+3 ≤ 8n (n > 3) ✅ 예제 2: f(n) = 3n³ + 7n + 5 → O(n³) n > 1일 때: 3n³ + 7n + 5 ≤ (3+7+5)n³ = 15n³ ∴ c=15, n₀=1 ✅ 핵심 규칙: 최고차항의 차수만 남기고 계수·상수는 무시!

복잡도 성장 속도 순위 (느린 → 빠름)

표기이름예시 알고리즘n=1000일 때
O(1)상수배열 인덱스 접근1
O(log n)로그Binary Search~10
O(n)선형Linear Search1,000
O(n log n)선형로그Merge Sort~10,000

⚠️ O(2ⁿ)이나 O(n!) 알고리즘은 n=50만 되어도 현실적으로 실행 불가능합니다!

Big-Θ(Theta)가 Big-O, Big-Ω보다 더 "정확한" 이유는?
✅ Θ(g(n)) = O(g(n)) AND Ω(g(n)). 위아래 모두 같은 g로 샌드위치처럼 묶기 때문에 가장 정확!
