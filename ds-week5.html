반복(Iterative)

bool search_iterative( Node* head, int key) { Node *cur = head; while (cur != NULL) { if (cur->element == key) return true; cur = cur->next; } return false; // 못 찾음 }

재귀(Recursive)

bool search_recursive( Node* head, int key) { // Base case: 리스트 끝 if (head == NULL) return false; // 현재 노드 확인 if (head->element == key) return true; // 나머지 재귀 탐색 return search_recursive( head->next, key); }
시간복잡도: O(n) — 최악의 경우 모든 노드 방문. 배열의 O(1) Random Access와 달리 순차 탐색만 가능!

② 중간 / 마지막 노드 삭제 — 통합 코드

prev_node와 cur_node 두 포인터를 함께 이동하며 key를 탐색. 발견 시 prev의 next를 cur의 next로 연결하여 cur를 우회한 뒤 해제.

void deleteNode(Node** head_ref, int key) { if (*head_ref == NULL) { printf("List is Empty.\n"); return; } // Case 1: 첫 번째 노드가 key if ((*head_ref)->element == key) { Node *temp = *head_ref; *head_ref = (*head_ref)->next; free(temp); return; } // Case 2 & 3: 중간 또는 마지막 노드 Node *prev_node = *head_ref; Node *cur_node = (*head_ref)->next; while (cur_node->next != NULL && cur_node->element != key) { prev_node = cur_node; cur_node = cur_node->next; } if (cur_node->element != key) { printf("Deletion failed: No key in the list.\n"); return; } Node *temp = cur_node; prev_node->next = cur_node->next; // 우회 연결! free(temp); // 메모리 해제 }
순서 중요: prev_node->next = cur_node->next 연결 후에 free(temp). 먼저 free하면 cur_node->next 주소를 잃는다!

③ 전체 리스트 삭제 (Delete Entire List)

단순히 head = NULL로 설정해도 메모리는 해제되지 않는다! 각 노드를 하나씩 순회하며 직접 해제해야 한다.

1
현재 노드의 next 주소를 next_node에 먼저 저장
2
free(cur_node) — 현재 노드 해제
3
cur_node = next_node — 다음으로 이동, NULL까지 반복
void deleteList(struct Node** head_ref) { Node* cur_node = *head_ref; Node* next_node; while (cur_node != NULL) { next_node = cur_node->next; // ① next 먼저 저장 free(cur_node); // ② 현재 해제 cur_node = next_node; // ③ 다음으로 이동 } *head_ref = NULL; // head 초기화 }
next를 먼저 저장하지 않고 free()하면 다음 노드 주소를 잃어 나머지 노드를 영원히 해제할 수 없다 → Memory Leak!

Insertion at Beginning (맨 앞 삽입)

단순 LL: new→next = head, head = new (2단계). DLL: 기존 head의 prev도 업데이트해야 한다.

1
새 노드 생성
2
new_node->next = *head_ref, new_node->prev = NULL
3
(*head_ref)->prev = new_node (기존 head의 prev → 새 노드), *head_ref = new_node
포인터 업데이트 4개: new→next, new→prev, 기존head→prev, head 자체. 단순 LL의 2개보다 복잡!

Insertion at End (맨 끝 삽입)

1
새 노드 생성
2
new_node->prev = tail (현재 tail을 prev로), new_node->next = NULL
3
tail->next = new_node (기존 tail의 next → 새 노드)
새 노드의 prev가 기존 tail을 가리키고, 기존 tail의 next가 새 노드를 가리킨다 → 양방향 연결 완성!

Insertion at Middle (중간 삽입)

position 노드 뒤에 삽입. 4개의 포인터를 정확한 순서로 업데이트해야 한다.

1
새 노드 생성
2
new_node->prev = position, new_node->next = position->next
3
position->next = new_node, position->next->prev = new_node ← 뒤 노드의 prev도 업데이트!
순서 필수: step 3에서 position->next = new_node를 먼저 하면, position->next->prev가 이미 new_node를 가리키므로 오류. step 2를 완전히 마친 후 step 3 진행!

Deletion in DLL — 3가지 경우 비교

케이스핵심 추가 단계 (단순 LL과의 차이)
첫 번째 노드 temp = head → head = head→next → head→prev = NULL → free(temp)
마지막 노드 traverse → temp = tail → prev 노드→next = NULL (새 tail의 next를 NULL로) → free(temp)
중간 노드 temp = target → prev→next = temp→next → temp→next→prev = prev → free(temp)
DLL 삭제의 핵심 차이: 단순 LL은 prev→next만 연결. DLL은 반드시 next→prev도 업데이트해야 양방향 일관성이 유지된다!

Linked List vs Array — 완전 비교

시간복잡도 비교

연산ArrayLinked List
Traverse & Search O(n) O(n)
Insertion (임의 위치) O(n) — shift 필요 O(1) — 포인터만 변경
Insertion (끝) O(1) O(1)
Deletion (임의 위치) O(n) — shift 필요 O(1) — 포인터만 변경
Random Access O(1) — 인덱스 직접 접근 불가 — 순차 탐색만
삽입/삭제 O(1)은 삽입/삭제 위치를 이미 알 때의 복잡도. 위치를 먼저 탐색해야 한다면 O(n) + O(1) = O(n).

장단점 비교

📦 Array

✅ 단순하고 다루기 쉬움
✅ Random Access O(1)
✅ 연속 메모리 → 캐시 효율 좋음

❌ 크기 고정 — 예측 불가 입력 처리 어려움
❌ 삽입/삭제 비용 큼 (shift)

🔗 Linked List

✅ 동적 크기 — 런타임에 조절
✅ 삽입/삭제 효율적 (O(1))

❌ 포인터 저장 → 추가 메모리 사용
❌ Random Access 불가
❌ 비연속 메모리 → 캐시 성능 나쁨
❌ 정렬되어도 이진 탐색 불가

선택 기준:
크기 고정 + 빠른 접근 필요 → Array
크기 가변 + 잦은 삽입/삭제 → Linked List

캐시 성능 문제 (Poor Cache Performance)

CPU는 메모리를 블록(캐시 라인) 단위로 가져온다. 배열은 연속된 메모리이므로 arr[0]을 읽을 때 arr[1], arr[2]…도 함께 캐시에 올라온다 → 캐시 히트율 높음.

연결 리스트는 노드가 메모리 여기저기에 흩어져 있어, 각 노드에 접근할 때마다 캐시 미스가 발생할 수 있다.

연결 리스트는 정렬되어 있어도 이진 탐색 불가 — 중간 노드로 바로 점프할 수 없기 때문. Skip List가 이 문제를 해결!

응용 — 다항식 덧셈 (Polynomial Addition)

개념

두 다항식에서 같은 지수(exponent)를 가진 항끼리 계수(coefficient)를 더한다.

(5x100 + 7x10 + x2 + 5) + (8x55 − 4x10 + x5 + x2 − 3)
= 5x100 + 8x55 + 3x10 + x5 + 2x2 + 2

x10 항: 7 + (−4) = 3  |  x2 항: 1 + 1 = 2  |  상수: 5 + (−3) = 2

Array vs Linked List 구현 비교

📦 Array 구현

typedef struct { // Poly[expon]=coefficient int coeffList[MAX_DEGREE+1]; int pow; } Polynomial; // 5x^100 + 7x^10 + x^2 + 5 // Poly[100]=5, Poly[10]=7 // Poly[2]=1, Poly[0]=5 // 나머지 97개 칸은 0...
최고 차수 100 → 배열 크기 101 필요. 항이 4개뿐이어도 97칸 낭비!

🔗 Linked List 구현

typedef struct Node { int coeff; // 계수 int pow; // 지수 struct Node* next; } Polynomial; // 실제 항만 노드로! // Head → [5|100] // → [7|10] // → [1|2] // → [5|0] → NULL
항이 적으면 메모리 절약. 희소 다항식(sparse polynomial)에 적합!

polyadd() — 덧셈 알고리즘

두 다항식 리스트를 동시에 순회(Two-pointer)하며 지수를 비교. 3가지 경우를 처리한다.

경우조건처리
Case 1poly1 지수 > poly2 지수poly1 항을 결과에 추가, poly1만 advance
Case 2poly1 지수 < poly2 지수poly2 항을 결과에 추가, poly2만 advance
Case 3지수 동일계수 합산 후 결과에 추가, 둘 다 advance
void polyadd(Node* poly1, Node* poly2, Node* result) { while (poly1->next && poly2->next) { if (poly1->pow > poly2->pow) { // Case 1 result->pow = poly1->pow; result->coeff = poly1->coeff; poly1 = poly1->next; } else if (poly1->pow < poly2->pow) { // Case 2 result->pow = poly2->pow; result->coeff = poly2->coeff; poly2 = poly2->next; } else { // Case 3: 지수 같음 result->pow = poly1->pow; result->coeff = poly1->coeff + poly2->coeff; // 계수 합산! poly1 = poly1->next; poly2 = poly2->next; } // 다음 결과 노드 동적 생성 result->next = (Node*)malloc(sizeof(Node)); result = result->next; result->next = NULL; } // 한쪽 리스트가 남은 경우 → while(poly1->next) 또는 while(poly2->next)로 나머지 추가 }
시간복잡도: O(m + n) (m, n = 각 다항식의 항 수). Case 3에서 둘 다 advance하지 않으면 같은 지수 항을 중복 처리하는 버그 발생!

Skip List

탄생 배경 — 연결 리스트의 탐색 문제 해결

정렬된 배열은 이진 탐색으로 O(log n). 하지만 연결 리스트는 정렬되어 있어도 임의 위치 접근이 불가 → O(n) 탐색.

Perfect Skip List 구조

각 상위 레벨은 바로 아래 레벨의 절반(½) 노드만 포함.

레벨포함 노드 수특징
Level 0 (맨 아래)n개모든 노드 — 정렬된 연결 리스트
Level 1n/2 개짝수 인덱스 노드만
Level 2n/4 개4의 배수 인덱스만
Level kn/2k 개…
n개 노드 → 약 log₂n 레벨. 각 레벨에서 최대 2번 비교 이동 → 전체 탐색: O(log n)

탐색 규칙 — 3가지

최상위 레벨 맨 왼쪽(-∞)에서 시작. 현재 노드의 key를 목표 k와 비교.

조건의미행동
k == key현재 노드가 목표✅ 탐색 완료!
k < key현재 노드 > 목표 (오른쪽으로 가면 더 커짐)⬇️ 한 레벨 아래로 내려감
k ≥ key현재 노드 ≤ 목표 (더 오른쪽에 있을 수 있음)➡️ 오른쪽으로 이동

예: Search(71) 과정

Lv3: -∞ 시작 → 71 ≥ -∞? → 오른쪽 → 31 도착 Lv3: 71 ≥ 31? Yes → 오른쪽 → Inf 도착 Lv3: 71 < Inf → 아래로 ↓ (Lv2로) Lv2: 31에서 → 71 ≥ 31? Yes → 오른쪽 → 91 Lv2: 71 < 91 → 아래로 ↓ (Lv1로) Lv1: 31에서 → 오른쪽으로 이동하며 탐색... Lv0: 71 발견! ✅
각 레벨에서 범위를 절반으로 좁히는 방식 → 이진 탐색과 유사한 O(log n) 성능!

Skip List = Linked List의 삽입/삭제 O(1) 장점 + 탐색을 O(n) → O(log n)으로 개선. 단, 추가 포인터를 위한 메모리 필요 (각 노드가 여러 레벨의 next 포인터를 가짐).
