What is a Stack?

Core Concept
A Stack is an ordered list where all insertions and deletions happen at one end, called the top. It follows LIFO — Last-In First-Out, just like a stack of plates: you always pick up the top one.
Real-life analogy
Think of a pile of pancakes 🥞. You always add a new pancake on top, and always eat the top one first. You can't grab one from the middle without moving all the ones above it!

Stack ADT (Abstract Data Type)

Stack Operations
OperationDescriptionEdge Case
Create(capacity)Create empty stack of given size—
Push(stack, item)Insert item at the topError if isFull
Pop(stack)Remove & return top itemError if isEmpty
Peek(stack)Return top item without removingError if isEmpty
isFull(stack)Returns true if top == capacity-1—
isEmpty(stack)Returns true if top == -1—
⚠️ Common Pitfalls
Stack Overflow: Pushing to a full stack → always check isFull() before Push.
Stack Underflow: Popping from an empty stack → always check isEmpty() before Pop.
An empty stack has top = -1; a full stack has top = capacity - 1.

Array Implementation

The stack struct stores three things: the top index, the capacity, and a pointer to the array.

// Stack type definition
typedef struct {
    int top;
    int capacity;
    int *array;  // can be char, float, double, etc.
} Stack;

Stack *createStack(int capacity) {
    Stack *stack = (Stack*)malloc(sizeof(Stack));
    stack->top = -1;          // empty: top starts at -1
    stack->capacity = capacity;
    stack->array = (int*)malloc(capacity * sizeof(int));
    return stack;
}

int isFull(Stack *stack)  { return stack->top == stack->capacity - 1; }
int isEmpty(Stack *stack) { return stack->top == -1; }
int peek(Stack *stack)    { return stack->array[stack->top]; }

void push(Stack *stack, int item) {
    if (!isFull(stack))
        stack->array[++stack->top] = item;   // pre-increment top, then insert
    else printf("Stack is full!\n");
}

int pop(Stack *stack) {
    if (!isEmpty(stack))
        return stack->array[stack->top--];   // return top, then post-decrement
    printf("Stack is Empty!\n");
    return 0;
}
💡 Key insight: pre vs post increment
Push uses ++stack->top (increment first, then assign) so the first element goes to index 0, not -1.
Pop uses stack->top-- (return first, then decrement) so you get the current top before moving down.

Stack Example: Palindromes

What is a Palindrome?

A word/phrase that reads the same forwards and backwards. Examples: mom, level, civic, radar.

Palindrome ABCDCBA ↔ ABCDCBA (same reversed)

Not Palindrome ABDA ↛ ADBA (different reversed)

Algorithm using Stack
  1. Push the first length/2 characters onto the stack
  2. If the string length is odd, skip the middle character (i++)
  3. For the remaining characters, Pop from stack and compare with current character
  4. If any mismatch → NOT a palindrome. If all match → IS a palindrome
int isPalindrome(Stack *stack, char word[], int length) {
    // Phase 1: push first half
    for (int i = 0; i < length / 2; i++)
        push(stack, word[i]);

    int i = length / 2;
    if (length % 2 != 0) i++;  // skip middle char if odd length

    // Phase 2: compare with popped chars
    while (i < length) {
        if (pop(stack) != word[i])
            return 0;  // mismatch → not a palindrome
        i++;
    }
    return 1;  // all matched → palindrome!
}

Stack Example: Bracket Check

Given an expression like [{( )}][](), check if all brackets are balanced.

Algorithm: Scan left to right. If you see an opening bracket → Push. If you see a closing bracket → Pop and verify it matches. After scanning, if the stack is empty → balanced!

Stack Example: Postfix Expression

Three Notation Types
NotationPatternExample
Infix (human)operand op operandA + B
Prefix (Polish)op operand operand+ A B
Postfix (compiler)operand operand opA B +
Why Postfix?
Simple to parse · No need for brackets · Not affected by operator precedence · Efficiently evaluated using a Stack
Example: Infix a - b * d is ambiguous without knowing precedence, but Postfix a b d * - is unambiguous!
Evaluating Postfix: 5 3 2 * + 8 -

Algorithm: scan left-to-right → if operand: push → if operator: pop 2, compute, push result

Converting Infix → Postfix

Rule: scan left to right.

  • If operand: output it directly
  • If operator: push if stack empty or higher precedence; else pop first
  • If '(': push to stack
  • If ')': pop until matching '(' is found

What is a Queue?

Core Concept
A Queue is a list where insertion happens at one end (tail/rear) and deletion at the other end (head/front). It follows FIFO — First-In First-Out, just like a real queue of people waiting in line!
Real-life analogy
Think of a ticket line at a movie theater 🎬. People join at the back and are served from the front. The first person who joined is the first to get their ticket!

Array Implementation

typedef struct {
    int front, rear;
    int capacity;
    int *array;
} Queue;

Queue *createQueue(int capacity) {
    Queue *q = (Queue*)malloc(sizeof(Queue));
    q->front = 0;
    q->rear  = -1;       // empty queue: rear starts at -1
    q->capacity = capacity;
    q->array = (int*)malloc(capacity * sizeof(int));
    return q;
}

int isFull(Queue *q)  { return q->rear == q->capacity - 1; }
int isEmpty(Queue *q) { return q->front == q->rear; } // Wait! see linear queue issue!

void enqueue(Queue *q, int item) {
    if (!isFull(q))
        q->array[++q->rear] = item;  // rear advances right
    else printf("Queue is Full!\n");
}

int dequeue(Queue *q) {
    if (!isEmpty(q))
        return q->array[q->front++];  // front advances right
    printf("Queue is Empty!\n");
    return -1;
}

Problem: Linear Queue Limitation

⚠️ The "Ghost Space" Problem
In a linear queue with capacity 5, after several Enqueue/Dequeue operations, front advances right. Even if the left slots are empty, rear == capacity-1 triggers "Queue is Full!" — wasting memory! This is called ghost space.

Solution: Circular Queue

Circular Queue
Arrange elements in a circle. Instead of advancing linearly, use modulo arithmetic: rear = (rear + 1) % capacity. This lets the rear "wrap around" to reuse freed front slots. A size variable tracks how many elements are currently stored. 
// Circular Queue: key changes vs linear
void enqueue(Queue *q, int item) {
    if (!isFull(q)) {
        q->rear = (q->rear + 1) % q->capacity;  // wrap around!
        q->array[q->rear] = item;
        q->size += 1;
    }
}

int dequeue(Queue *q) {
    if (!isEmpty(q)) {
        int element = q->array[q->front];
        q->front = (q->front + 1) % q->capacity; // wrap around!
        q->size -= 1;
        return element;
    }
    return -1;
}

int isFull(Queue *q)  { return q->size == q->capacity; }
int isEmpty(Queue *q) { return q->size == 0; }
💡 The key difference
Linear queue uses front == rear to detect empty (but this conflicts with "one element left"). Circular queue solves this cleanly by tracking size: isFull = size==capacity, isEmpty = size==0.

What is a Linked List?

Core Concept
A Linked List is a collection of nodes where each node contains an element (data) and a pointer to the next node. Unlike arrays, nodes are NOT necessarily adjacent in memory — they can be scattered anywhere, connected only by pointers.
Why Use a Linked List?
AdvantageExplanation
Dynamic sizeAllocate/deallocate memory at runtime — no fixed size like arrays
Easy insertion/deletionJust redirect pointers — no shifting elements like in arrays
Efficient memoryOnly allocate memory when you actually need it
⚠️ Trade-offs
No random access (can't do list[3] — must traverse). Extra memory for pointers. Cache-unfriendly because nodes are scattered in memory.

Basic Operations

Three fundamental operations: Traversal Search Insert / Delete

1. Traversal

Follow the pointer of each node until you reach NULL. Process each node's element along the way.

void traverse(Node *head) {
    Node *cur_node = head;
    while (cur_node != NULL) {
        printf("%d ", cur_node->element);
        cur_node = cur_node->next;  // follow the pointer
    }
}

2. Search

// Iterative search
bool search_iterative(Node *head, int key) {
    Node *cur_node = head;
    while (cur_node != NULL) {
        if (cur_node->element == key) return true;
        cur_node = cur_node->next;
    }
    return false;
}

// Recursive search
bool search_recursive(Node *head, int key) {
    if (head == NULL) return false;           // base case: not found
    if (head->element == key) return true;    // found!
    return search_recursive(head->next, key); // check rest of list
}

3. Insertion

Three positions to insert: Beginning End Middle

// Insert at beginning (O(1))
void insertAtHead(Node **head_ref, int new_data) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->element = new_data;
    new_node->next = (*head_ref);   // ② point to old head
    (*head_ref) = new_node;         // ③ update head to new node
}

// Insert at end (O(n) — must traverse)
void insertAtEnd(Node *head, int new_data) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->element = new_data;
    new_node->next = NULL;

    Node *last = head;
    while (last->next != NULL)      // ② traverse to last
        last = last->next;
    last->next = new_node;          // ③ link last node to new
}

// Insert after a specific key position (O(n))
void insertAfterPosition(Node *head, int new_data, int key) {
    Node* new_node = (Node*)malloc(sizeof(Node));
    new_node->element = new_data;

    Node *position = head;
    while (position->element != key) // ② find position
        position = position->next;

    new_node->next = position->next; // ③ new node points to what's after
    position->next = new_node;       // ④ position points to new node
}

4. Deletion

Three cases: Delete head Delete tail Delete intermediate

void deleteNode(Node **head_ref, int key) {
    if (*head_ref == NULL) { printf("List is Empty.\n"); return; }

    // Case 1: Delete the head node
    if ((*head_ref)->element == key) {
        Node *temp = *head_ref;
        *head_ref = (*head_ref)->next;  // move head to next
        free(temp);                      // free old head
        return;
    }

    // Case 2 & 3: Delete middle or last node
    Node *prev_node = *head_ref;
    Node *cur_node  = (*head_ref)->next;

    while (cur_node != NULL && cur_node->element != key) {
        prev_node = cur_node;
        cur_node  = cur_node->next;
    }

    if (cur_node == NULL) { printf("Key not found.\n"); return; }

    // Bypass the node to be deleted
    prev_node->next = cur_node->next;
    free(cur_node);  // same code handles both middle and tail!
}

// Delete entire list
void deleteList(Node **head_ref) {
    Node *cur_node = *head_ref;
    Node *next_node;
    while (cur_node != NULL) {
        next_node = cur_node->next;  // save next before freeing
        free(cur_node);
        cur_node = next_node;
    }
    *head_ref = NULL;
}
💡 Why double pointer (Node **)?
When we delete the head, we need to update the head variable itself in main(). A single pointer *head only lets us change what head points to, but **head_ref lets us change the actual head pointer. Think of it like passing an address of an address.
