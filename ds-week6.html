Tree 정의 & 용어

Tree란?

노드(node)들이 간선(edge)으로 연결된 비선형, 계층적 자료구조. 사이클(cycle)이 없으며, 하나의 루트 노드에서 시작해 자식 노드들이 뻗어나간다.

실생활 예: 파일 시스템 (C: → Program Files / Users → App1, App2...), 웹 페이지의 DOM 트리, HTML 태그 구조.

트리의 종류

• Binary Tree — 자식 최대 2개
• Ternary Tree — 자식 최대 3개
• N-ary Tree — 자식 최대 N개

핵심 성질

• 각 노드까지의 경로는 유일
• 간선 수 = 노드 수 − 1
• 재귀적 자료구조 (서브트리의 집합)

핵심 용어 정리

Node (노드)
데이터와 다른 노드로의 링크를 저장하는 기본 단위
Edge (간선)
두 노드(부모→자식)를 연결하는 선. n개 노드 → n-1개 간선
Root (루트)
부모가 없는 최상위 노드. 예: A
Parent & Child
P→Q 간선이 있으면 P가 부모, Q가 자식. 예: A는 B,C,D의 부모
Siblings (형제)
같은 부모를 가진 노드들. 예: E와 F (부모 = B)
Leaf (잎) / Internal
자식이 없으면 Leaf. 자식이 있으면 Internal(내부) 노드
Path (경로)
시작→목적지까지 노드의 순서. 예: A→J = <A,B,F,J>
Ancestor & Descendant
Root→Q 경로에 P가 있으면: P는 Q의 조상, Q는 P의 자손
Degree (차수)
노드의 차수 = 자식 수 (D의 차수 = 2). 트리의 차수 = 최대 노드 차수
Level / Depth (레벨)
Root까지 경로의 길이. Root = Level 0. 트리의 레벨 = 최대 레벨. 예: 4
Width (너비)
같은 레벨의 노드 수 합 중 최댓값. 예: Level 2에 5개 → Width = 5
Size (크기)
트리의 전체 노드 수. 예: 11
Subtree (서브트리)
노드 P를 루트로 하는 서브트리 = P 자신 + P의 모든 자손 노드. 예: D를 루트로 하는 서브트리 = {D, H, I}
핵심 성질 3가지: ①각 노드까지 경로는 유일(unique) ②간선 수 = 노드 수 - 1 ③재귀적 구조 — 모든 서브트리 자체도 트리

왜 Tree를 쓰는가?

특징설명
계층적 구조파일 시스템, 조직도 등 계층 관계를 자연스럽게 표현
효율적 탐색BST 기준 평균 O(log n) 탐색
효율적 삽입/삭제탐색 시간에 추가 비용 없이 삽입/삭제 가능
동적 크기필요에 따라 동적으로 커지거나 줄어듦

Binary Tree (이진 트리)

정의

모든 노드가 0, 1, 또는 2개의 자식을 갖는 트리. 각 자식은 왼쪽(left) 또는 오른쪽(right)으로 구분된다.

typedef struct tNode { int data; struct tNode* left_child; struct tNode* right_child; } tNode;
연결 리스트로 구현 시 각 노드가 left_child, data, right_child 세 필드를 가진다. Complete Binary Tree는 배열로도 표현 가능!

배열 표현 (Complete Binary Tree)

완전 이진 트리는 배열로 나타낼 수 있다. 인덱스 i인 노드에 대해:

관계인덱스 공식조건
부모parent(i) = ⌊i/2⌋ - 1-
왼쪽 자식leftChild(i) = 2i + 12i+1 < n
오른쪽 자식rightChild(i) = 2i + 22i+2 < n
예: i=1 (노드 B) → 왼쪽 자식 = 인덱스 3 (D), 오른쪽 자식 = 인덱스 4 (E)

이진 트리의 종류 — 4가지 비교

종류정의핵심 조건
Full Binary Tree
포화 이진 트리(엄격한)		 모든 노드가 자식 0개 또는 2개 자식이 1개인 노드 없음
Complete Binary Tree
완전 이진 트리		 마지막 레벨을 제외한 모든 레벨이 완전히 채워짐. 마지막 레벨은 왼쪽부터 채워짐 마지막 레벨 노드가 왼쪽부터
Balanced Binary Tree
균형 이진 트리		 모든 노드에서 왼쪽/오른쪽 서브트리 높이 차이 ≤ 1 높이 차 ≤ 1
Perfect Binary Tree
완전 포화 이진 트리		 모든 내부 노드가 2개 자식. 모든 Leaf가 같은 레벨 Full + 모든 Leaf 같은 깊이

이진 트리 수학적 성질

성질공식예시 (h=3)
레벨 i의 최대 노드 수2iLevel 3 → 최대 8개
높이 h인 트리의 최대 노드 수2h+1 − 1h=3 → 최대 15개
n개 노드 트리의 최소 높이log(n+1) − 1n=7 → 최소 높이 2
n개 노드 트리의 간선 수n − 17개 노드 → 6개 간선
Perfect Binary Tree: 높이 h → 정확히 2h+1−1개 노드. h=3이면 15개. 반대로 n=15이면 h=log(15+1)−1=3.

Tree Traversal (트리 순회)

4가지 순회 방법

순회순서분류대표 활용
In-orderLeft → Root → RightDFS정렬된 순서로 값 출력
Pre-orderRoot → Left → RightDFS트리 복사본 생성
Post-orderLeft → Right → RootDFS리프부터 트리 삭제
Level-order레벨 순서(위→아래, 좌→우)BFS레벨 단위 처리
암기법: In-order = 루트가 중간, Pre-order = 루트가 먼저, Post-order = 루트가 마지막

Binary Search Tree (이진 탐색 트리)

정의 & 핵심 성질

이진 트리이면서 다음 조건을 만족하는 트리:

왼쪽 서브트리

현재 노드보다 작은 값만 포함

오른쪽 서브트리

현재 노드보다 큰 값만 포함

최솟값 = 가장 왼쪽 노드 (10), 최댓값 = 가장 오른쪽 노드 (95). In-order 순회하면 10→15→20→25→30→…→95 정렬된 순서로 출력!

Search(x, T) — 탐색

루트에서 시작해 x와 현재 노드 값을 비교하며 반복적으로 범위를 절반씩 좁혀간다.

조건행동
x == 현재 노드✅ 찾음! 포인터 반환
x < 현재 노드왼쪽 서브트리로 재귀
x > 현재 노드오른쪽 서브트리로 재귀
NULL 도달❌ 없음, NULL 반환
BinaryTreeNode* searchNode(BinaryTreeNode* root, int target) { if (root == NULL) return NULL; // 못 찾음 if (root->key > target) return searchNode(root->left, target); // 왼쪽으로 else if (root->key < target) return searchNode(root->right, target); // 오른쪽으로 else return root; // 찾음! }

예: Search(30, T) → 50보다 작음 → 25로 이동 → 25보다 큼 → 35로 이동 → 35보다 작음 → 30 발견!

Insert(x, T) — 삽입

Search와 동일한 방식으로 내려가다가 NULL을 만나면 그 자리에 새 노드를 삽입. 예: Insert(65, T)

50 → 65>50, 오른쪽 → 75 → 65<75, 왼쪽 → 60 → 65>60, 오른쪽 → 70 → 65<70, 왼쪽 자리가 NULL → 65 삽입!

BinaryTreeNode* insertNode(BinaryTreeNode* node, int value) { if (node == NULL) { // NULL 자리 = 삽입 위치 BinaryTreeNode *temp = malloc(sizeof(BinaryTreeNode)); temp->key = value; temp->left = temp->right = NULL; return temp; } if (value < node->key) node->left = insertNode(node->left, value); // 왼쪽으로 else if (value > node->key) node->right = insertNode(node->right, value); // 오른쪽으로 // else: 이미 있는 값 → 아무것도 안 함 return node; }
삽입은 항상 Leaf 자리에 발생한다. 기존 구조를 건드리지 않고 빈 자리에만 연결하면 된다!

Delete(x, T) — 삭제 3가지 경우

경우처리법예시
① 자식 없음 (Leaf) 그냥 삭제! 28 삭제 → 그냥 제거
② 자식 1개 삭제 후 자식을 그 자리에 연결 (plug in) 35 삭제 (자식: 30) → 30이 35 자리로
③ 자식 2개 오른쪽 서브트리의 최솟값(in-order successor)으로 대체 후 그 노드 삭제 25 삭제 → 오른쪽 서브트리 최솟값 = 28 → 28이 25 자리로
Case ③ 주의: 대체 노드(in-order successor)는 항상 오른쪽 서브트리의 가장 왼쪽 노드. 대체 후 원래 위치에서 그 노드도 삭제해야 한다.

BST 성능 분석

경우트리 형태탐색/삽입/삭제
Best caseBalanced / Complete Binary TreeO(log n)
Worst caseSkewed Binary Tree (한쪽으로만 치우침)O(n)
Average case일반적인 랜덤 입력O(log n)
Skewed(편향) 트리: 1, 2, 3, 4, 5 순서로 삽입하면 오른쪽으로만 뻗어 연결 리스트와 동일해진다 → O(n). AVL Tree, Red-Black Tree로 균형을 강제 유지!
BST 활용: 빠른 조회(평균 O(log n)), 정렬된 데이터 저장, 자동완성(사전), 심볼 테이블(컴파일러). In-order 순회로 항상 정렬된 출력 가능!
