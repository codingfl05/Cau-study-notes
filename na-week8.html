Numerical Analysis · Week 8

곡면 표현 (Surfaces)

Bilinear · Bi-cubic · Bezier · Spline 곡면 + 렌더링 & 세분화

왜 곡면을 수학적으로 표현하는가?

자동차, 비행기, 주전자 같은 3D 물체의 경계를 정확하게 컴퓨터로 표현하려면 수학적 곡면 모델이 필요합니다.

폴리곤 메쉬 (Polygonal Mesh)
삼각형/사각형 조각들로 곡면을 근사. 빠르지만 꺾임이 보임.
서피스 패치 (Surface Patch)
수학 함수로 정의한 부드러운 곡면 조각. 품질은 좋지만 계산이 더 필요.

곡면 표현 방식

① 음함수 표현

f(x, y, z) = 0 예시: - 평면: ax + by + cz + d = 0 - 구: x² + y² + z² - r² = 0

② 매개변수 표현 (Parametric)

곡선에서 매개변수가 u 하나였다면, 곡면은 u와 v 두 개가 필요합니다!

p(u, v) = [x(u,v), y(u,v), z(u,v)]ᵀ (0 ≤ u ≤ 1, 0 ≤ v ≤ 1)
🗺️ 비유: u를 "경도", v를 "위도"라고 생각하세요. (u, v) 쌍이 하나의 좌표처럼 3D 공간의 한 점을 가리킵니다.

피스와이즈 패치 (Piecewise Patches)

큰 곡면을 작은 조각(patch)들로 나누어 표현합니다. 각 패치는 u, v가 [0,1] 범위를 가집니다.

Bilinear 곡면

💡 핵심 아이디어: 4개의 꼭짓점(corner points)을 이용해 선형 보간으로 곡면을 만듭니다. 가장 단순한 곡면!

직관적 이해

먼저 v 방향으로 선형 보간해서 양쪽 가장자리 점을 구하고, 그 두 점을 u 방향으로 다시 선형 보간합니다.

수식 전개

Step 1: v 방향으로 먼저 보간
P₀,ᵥ = (1-v)·P₀,₀ + v·P₀,₁ (왼쪽 가장자리) P₁,ᵥ = (1-v)·P₁,₀ + v·P₁,₁ (오른쪽 가장자리)
Step 2: u 방향으로 두 점을 보간
P(u,v) = (1-u)·P₀,ᵥ + u·P₁,ᵥ
Step 3: 전개하면
P(u,v) = (1-u)(1-v)·P₀,₀ + u(1-v)·P₁,₀ + (1-u)v·P₀,₁ + uv·P₁,₁

직접 계산 예제

꼭짓점: P₀,₀=(0,0,0), P₁,₀=(4,0,0), P₀,₁=(0,3,1), P₁,₁=(4,3,2)

u=0.5, v=0.5일 때 P(0.5, 0.5) = ?

계산
(1-0.5)(1-0.5)=0.25, u(1-v)=0.25, (1-u)v=0.25, uv=0.25

x: 0.25×0 + 0.25×4 + 0.25×0 + 0.25×4 = 2.0
y: 0.25×0 + 0.25×0 + 0.25×3 + 0.25×3 = 1.5
z: 0.25×0 + 0.25×0 + 0.25×1 + 0.25×2 = 0.75
→ P(0.5, 0.5) = (2.0, 1.5, 0.75)

장단점

✅ 장점
4개 꼭짓점만 필요. 계산이 매우 단순.
❌ 단점
경계선이 직선. 곡면이 평평한 경향. 복잡한 모양 불가.

Bi-cubic 패치

💡 핵심 아이디어: 곡선을 3차 다항식으로 표현했듯이, 곡면을 u와 v 방향 모두 3차로 표현합니다.

수학적 표현

P(u,v) = Σᵢ Σⱼ aᵢⱼ · uⁱ · vʲ (i, j = 0, 1, 2, 3) 행렬 형태: P(u,v) = [1 u u² u³] · A · [1 v v² v³]ᵀ 여기서 A는 4×4 계수 행렬 (aᵢⱼ들로 구성)

이 식에 16개의 aᵢⱼ 계수가 있으므로 → 16개의 방정식이 필요!

16개 조건은 어떻게 결정?

Hermite 방식을 2D로 확장합니다. 4 꼭짓점에서 세 종류의 조건을 씁니다:

조건 종류개수내용
위치값4개P(0,0), P(0,1), P(1,0), P(1,1)
u 방향 편미분4개Pᵤ(0,0), Pᵤ(0,1), Pᵤ(1,0), Pᵤ(1,1)
v 방향 편미분4개Pᵥ(0,0), Pᵥ(0,1), Pᵥ(1,0), Pᵥ(1,1)
비틀림 벡터4개Pᵤᵥ(0,0), Pᵤᵥ(0,1), Pᵤᵥ(1,0), Pᵤᵥ(1,1)

합계: 4+4+4+4 = 16개 ✓

⚠️ 비틀림 벡터(Twist Vector) Pᵤᵥ란? u와 v를 동시에 미분한 값. 직관적으로 이해하거나 설정하기 매우 어렵습니다 — 이것이 Bi-cubic의 가장 큰 단점!

행렬 표현 (Hermite 블렌딩 함수 사용)

P(u,v) = [F₁(u) F₂(u) F₃(u) F₄(u)] · Q · [F₁(v) F₂(v) F₃(v) F₄(v)]ᵀ 여기서 Q는 16개 조건값(위치+편미분+비틀림)으로 구성된 4×4 행렬 Hermite 블렌딩 함수: F₁(u) = 1 - 3u² + 2u³ (시작점 위치) F₂(u) = 3u² - 2u³ (끝점 위치) F₃(u) = u - 2u² + u³ (시작 접선) F₄(u) = -u² + u³ (끝 접선)
✅ 장점: 경계곡선이 Hermite 곡선. 내부 형상 제어 가능.
❌ 단점: 비틀림 벡터 설정이 어려움.

Bezier 곡면

💡 핵심 아이디어: Bezier 곡선을 2D로 확장! 곡선의 4개 제어점 → 곡면의 4×4 = 16개 제어점

제어점 배열 (4×4 격자)

수식

P(u,v) = Σᵢ₌₀ⁿ Σⱼ₌₀ᵐ Pᵢ,ⱼ · Bᵢ,ₙ(u) · Bⱼ,ₘ(v) (0 ≤ u ≤ 1, 0 ≤ v ≤ 1) 직관적 이해: 1. v를 고정 → v 방향으로 Bezier 보간 → u 방향 제어점들을 구함 2. 그 제어점들로 u 방향 Bezier 곡선 → 곡면 위의 점

중요한 성질 검증: P(0,0) = P₀,₀

u=0, v=0 대입 시 증명
P(0,0) = Σᵢ Σⱼ Pᵢ,ⱼ · Bᵢ,ₙ(0) · Bⱼ,ₘ(0) Bernstein: Bₖ,ₙ(0) = 1 (k=0일 때만), 나머지 = 0 ∴ P(0,0) = P₀,₀ ✓ (모서리 점을 정확히 통과!)

경계선이 Bezier 곡선임을 증명: u=0 대입

P(0,v) = Σⱼ₌₀ᵐ P₀,ⱼ · Bⱼ,ₘ(v) → 이것은 정확히 {P₀,₀, P₀,₁, ..., P₀,ₘ}을 제어점으로 하는 v 방향 Bezier 곡선!
✅ Bezier 곡면의 장점:
1. 경계선이 Bezier 곡선
2. 제어점 조작이 직관적
3. 미분도 같은 방법으로 계산 가능
❌ 단점: 국소 제어 불가!
제어점 하나를 움직이면 → 전체 곡면이 영향 받음. 세밀한 편집이 어렵습니다.

Spline 곡면

💡 핵심 아이디어: B-Spline 곡선을 2D로 확장! 16개 제어점으로 C² 연속성을 가진 부드러운 곡면 생성.

수식

p(u,v) = Σᵢ₌₀³ Σⱼ₌₀³ bᵢ(u) · bⱼ(v) · Pᵢⱼ 행렬 형태: p(u,v) = uᵀ · Mₛ · P · Mₛᵀ · v 여기서: u = [1 u u² u³]ᵀ v = [1 v v² v³]ᵀ P = 4×4 제어점 행렬 (Pᵢⱼ) Mₛ = (1/6)· ⎡ 1 4 1 0⎤ ⎢-3 0 3 0⎥ ⎢ 3 -6 3 0⎥ ⎣-1 3 -3 1⎦

Spline 곡면의 구조

곡면 비교 요약

방법 제어점 수 통과 여부 연속성 비틀림벡터
Bilinear 4 ✅ 꼭짓점 C⁰ 불필요
Bi-cubic (Hermite) 16조건 ✅ 꼭짓점 C¹ ⚠️ 필요 (어려움)
Bezier 16 (4×4) ✅ 꼭짓점만 C⁰ 기본 불필요
Spline 16 (4×4) ❌ 근사 C² 자동! 불필요

곡선/곡면 렌더링

💡 렌더링 아이디어: u(와 v)를 여러 값으로 샘플링하고, 인접 점들을 선분으로 연결합니다. 샘플이 많을수록 부드러워 보입니다!

효율적인 계산: Horner의 방법

p(u) = c₀ + uc₁ + u²c₂ + u³c₃ Horner 변환: p(u) = c₀ + u(c₁ + u(c₂ + uc₃)) → 곱셈 횟수가 줄어 더 빠름!
⚡ 비유: (((c₃·u + c₂)·u + c₁)·u + c₀) 처럼 안쪽부터 계산. 불필요한 u² 계산 없음.

Bezier 곡선 세분화 (Subdivision)

하나의 Bezier 곡선을 두 개의 더 작은 Bezier 곡선으로 분할하는 방법. 재귀적으로 반복하면 충분히 평평해집니다.

기하학적 세분화 공식

l₁ = (1/2)(p₀ + p₁) ← p₀과 p₁의 중점 r₂ = (1/2)(p₂ + p₃) ← p₂과 p₃의 중점 l₂ = (1/2)(l₁ + (p₁+p₂)/2) ← l₁과 중간점의 중점 r₁ = (1/2)(r₂ + (p₁+p₂)/2) ← r₂과 중간점의 중점 l₃ = r₀ = (1/2)(l₂ + r₁) ← 분할점 (중점)

언제 세분화를 멈추나?

평탄도 검사 (Flatness Test)
l₁, l₂가 l₀~l₃를 잇는 선분에서 얼마나 떨어져 있는지 측정.
거리가 임계값(threshold) 이하 → 충분히 평평 → 중단하고 직선으로 그리기

다른 곡선을 Bezier로 변환 후 세분화

변환 공식: q = Mʙ⁻¹ · M · p 보간법 → Bezier: Mʙ⁻¹ · Mᵢ = ⎡1 0 0 0 ⎤ ⎢1/3 2/3 0 0 ⎥ ⎢0 1/3 2/3 0 ⎦ (일부) ⎣0 0 0 1 ⎤ B-Spline → Bezier: Mʙ⁻¹ · Mₛ = ⎡1/6 4/6 1/6 0 ⎤ ⎢0 4/6 2/6 0 ⎥ ⎢0 2/6 4/6 0 ⎥ ⎣0 1/6 4/6 1/6⎦

곡면 세분화

2D 곡면을 4개의 서브패치로 나누는 절차:

Step 1
u = 0, 1/3, 2/3, 1에서 고정 → v 방향 4개 곡선을 세분화
Step 2
Step 1에서 생성된 8개 반쪽 곡선에 대해 v = 0, 1/3, 2/3, 1 고정 → u 방향 세분화
Step 3
4개의 서브패치로 분할 → 재귀 반복
Step 4: 렌더링
충분히 평탄해지면 GL_QUADS(사각형 폴리곤)으로 렌더링
