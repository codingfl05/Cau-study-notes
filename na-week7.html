Numerical Analysis · Week 7

곡선 표현 (Curves)

Cubic · Hermite · Bezier · B-Spline 완전 정리

곡선이란? — 두 가지 표현 방식

컴퓨터 그래픽에서 곡선을 수학적으로 표현하는 방법은 크게 두 가지입니다.

① 음함수 표현 (Implicit Representation)

곡선을 f(x, y) = 0 형태의 방정식으로 표현합니다.

직선: ax + by + c = 0 원: x² + y² - r² = 0 (반지름 r인 원) 3D 평면: ax + by + cz + d = 0 3D 구: x² + y² + z² - r² = 0
💡 음함수는 "이 점이 곡선 위에 있는가?"를 확인하기 쉽지만, 곡선 위의 점을 순서대로 나열하기 어렵습니다.

② 매개변수 표현 (Parametric Representation)

독립 변수 u에 따라 x, y, z 좌표가 결정됩니다. 마치 시간(u)에 따라 물체가 이동하는 궤적처럼 생각하면 됩니다.

x = x(u) y = y(u) z = z(u) → p(u) = [x(u), y(u), z(u)]ᵀ
🚗 비유: u를 "시간"이라고 생각하세요. u=0일 때 출발점, u=1일 때 도착점. 그 사이의 u 값마다 곡선 위의 위치가 결정됩니다.

매개변수 표현의 미분 = 접선 벡터

dp(u)/du = [dx(u)/du, dy(u)/du, dz(u)/du]ᵀ 방향 = 접선 방향 크기 = u 변화에 따른 속도

왜 다항식(Polynomial)을 쓸까?

좋은 곡선 표현이 갖춰야 할 조건들:

조건의미
국소 제어 (Local control)한 부분을 바꿔도 전체에 영향 최소화
매끄러움 (Smoothness)연결점에서 불연속이 없음
미분 가능 (Differentiable)접선 벡터를 계산할 수 있음
수치 안정성 (Stability)작은 입력 변화가 큰 출력 변화를 유발하지 않음
렌더링 용이성화면에 그리기 쉬움
결론: 다항식 x(u) = c₀ + c₁u + c₂u² + ... + cₙuⁿ이 이 조건들을 잘 만족합니다.

왜 3차(Cubic)?

고차 다항식의 문제
급격한 변화, 진동(Runge 현상), 계산 복잡
저차 다항식의 문제
표현력 부족, 복잡한 모양 불가
⚖️ 3차(Cubic)가 절충점! 충분한 표현력 + 계산 효율성 + 부드러운 곡선
p(u) = c₀ + c₁u + c₂u² + c₃u³ (0 ≤ u ≤ 1)

미지수: c₀, c₁, c₂, c₃ (x, y, z 각각) → 총 12개 미지수

여러 세그먼트로 긴 곡선 만들기

복잡한 곡선은 짧은 3차 곡선 조각(segment)들을 이어 붙여서 만듭니다.

⚠️ 단순히 이어 붙이면 연결점에서 꺾임(불연속 미분)이 생길 수 있습니다. → 이것이 Hermite, Bezier, B-Spline이 발전한 이유!

Cubic 보간법 (Interpolation)

핵심 아이디어: 4개의 점을 지나는 3차 곡선을 구한다. 1개의 점 = 3개의 방정식(x,y,z), 4점 = 12개 방정식 = 12개 미지수 → 풀 수 있다!

u 값 배분

4개의 점을 [0, 1] 구간에 균등하게 배치합니다:

p₀ = p(0) p₁ = p(1/3) p₂ = p(2/3) p₃ = p(1)

행렬 방정식으로 변환

4개 점의 방정식을 행렬로 묶으면:

p = A·c 여기서 A는 4×4 행렬 ⎡ 1 0 0 0 ⎤ ⎢ 1 1/3 (1/3)² (1/3)³ ⎥ ⎢ 1 2/3 (2/3)² (2/3)³ ⎥ ⎣ 1 1 1 1 ⎦
풀이 과정
p = A·c
→ A를 역행렬로 이항:
c = A⁻¹·p = Mᵢ·p

Mᵢ (보간 기하행렬):
Mᵢ = A⁻¹ = ⎡ 1 0 0 0 ⎤ ⎢ -5.5 9 -4.5 1 ⎥ ⎢ 9 -22.5 18 -4.5 ⎥ ⎣ -4.5 13.5 -13.5 4.5⎦

블렌딩 함수 (Blending Functions)

곡선을 4개의 제어점의 가중 합으로 표현합니다:

p(u) = b₀(u)·p₀ + b₁(u)·p₁ + b₂(u)·p₂ + b₃(u)·p₃ b₀(u) = -(9/2)(u - 1/3)(u - 2/3)(u - 1) b₁(u) = (27/2)u(u - 2/3)(u - 1) b₂(u) = -(27/2)u(u - 1/3)(u - 1) b₃(u) = (9/2)u(u - 1/3)(u - 2/3)
📌 각 bₖ(u)는 u=k/3일 때 1이고 나머지 점에서 0입니다. 즉 각 제어점이 자기 위치 근처에서만 강하게 영향을 미칩니다.

블렌딩 함수 그래프

4개 이상의 점: 세그먼트 분할

7개의 점이 있다면, 4개씩 묶어서 두 개의 세그먼트를 만듭니다:

1번 세그먼트: p₀, p₁, p₂, p₃ 사용 2번 세그먼트: p₃, p₄, p₅, p₆ 사용 (p₃는 공유)
⚠️ 연결점(p₃)에서 미분이 일치하지 않을 수 있어 꺾임이 생깁니다. 이게 문제!

Hermite 곡선

💡 핵심 아이디어: 보간법은 중간 점 2개를 씁니다. Hermite는 대신 양 끝점의 미분(접선 벡터)을 씁니다. → 연결점에서 매끄럽게!

왜 Hermite인가?

보간법의 문제: "점들의 위치만으로는 곡선의 방향을 제어하기 어렵다"

Hermite 해결책: 끝점의 위치 + 끝점의 접선벡터(방향) 제공 → 총 4개의 조건으로 3차 곡선 결정

4가지 조건

p(0) = p₀ (시작점 위치) p(1) = p₃ (끝점 위치) p'(0) = 접선벡터₀ (시작점 방향) p'(1) = 접선벡터₁ (끝점 방향)

행렬 방정식 유도

p(u) = c₀ + c₁u + c₂u² + c₃u³, p'(u) = c₁ + 2c₂u + 3c₃u² 를 조건에 대입:

p(0) = c₀ p'(0) = c₁ p(1) = c₀ + c₁ + c₂ + c₃ p'(1) = c₁ + 2c₂ + 3c₃ → q = A·c 형태로: A = ⎡1 0 0 0⎤ ⎢1 1 1 1⎥ ⎢0 1 0 0⎥ ⎣0 1 2 3⎦ → c = A⁻¹·q = Mₕ·q
Hermite 기하행렬 Mₕ
Mₕ = ⎡ 1 0 0 0⎤ ⎢ 0 0 1 0⎥ ⎢-3 3 -2 -1⎥ ⎣ 2 -2 1 1⎦

Hermite 블렌딩 함수

p(u) = uᵀ·Mₕ·q = b(u)ᵀ·q b₀(u) = 1 - 3u² + 2u³ (시작점 위치 가중치) b₁(u) = 3u² - 2u³ (끝점 위치 가중치) b₂(u) = u - 2u² + u³ (시작점 접선 가중치) b₃(u) = -u² + u³ (끝점 접선 가중치)
✅ 장점: 연결점에서 접선 벡터를 맞춰주면 C¹ 연속성(매끄러운 연결) 보장!
⚠️ 단점: 최적의 접선 벡터를 어떻게 설정할지 모를 수 있습니다. 데이터 점만 있을 때 어렵습니다.

손으로 푸는 예제

예제 조건
p(0) = (0, 0), p(1) = (1, 0), p'(0) = (1, 2), p'(1) = (1, -2)
Step 1: 조건 벡터 구성 (x성분만 계산)
q = [p(0)ₓ, p(1)ₓ, p'(0)ₓ, p'(1)ₓ] = [0, 1, 1, 1]
Step 2: c = Mₕ·q 계산 (x성분)
c₀ = 1·0 + 0·1 + 0·1 + 0·1 = 0
c₁ = 0·0 + 0·1 + 1·1 + 0·1 = 1
c₂ = -3·0 + 3·1 + (-2)·1 + (-1)·1 = 0
c₃ = 2·0 + (-2)·1 + 1·1 + 1·1 = 0
Step 3: x(u) = 0 + 1·u + 0·u² + 0·u³ = u
x성분은 단순히 u에 비례! y성분도 같은 방법으로 계산.

Bezier 곡선

💡 핵심 아이디어: Hermite는 접선벡터를 직접 입력해야 합니다. Bezier는 인접 점의 차이로 접선을 자동 추정합니다!

접선 추정 방법

시작 접선: p'(0) = 3(p₁ - p₀) / (1/3) = 3(p₁ - p₀) 끝 접선: p'(1) = 3(p₃ - p₂)
🎯 비유: p₁과 p₂는 "당기는 손잡이". 직접 곡선 위에 있지 않지만, 곡선의 방향을 조종합니다!

Bezier 행렬 유도

방정식 4개: p₀ = c₀ p₁ = c₀ + (1/3)c₁ p₂ = c₀ + (2/3)c₁ + (1/3)c₂ p₃ = c₀ + c₁ + c₂ + c₃ → c = Mʙ·p
Bezier 기하행렬 Mʙ
Mʙ = ⎡ 1 0 0 0⎤ ⎢-3 3 0 0⎥ ⎢ 3 -6 3 0⎥ ⎣-1 3 -3 1⎦

Bernstein 다항식 (Bezier 블렌딩 함수)

p(u) = b(u)ᵀ·p b₀(u) = (1-u)³ ← p₀의 영향력 b₁(u) = 3u(1-u)² ← p₁의 영향력 b₂(u) = 3u²(1-u) ← p₂의 영향력 b₃(u) = u³ ← p₃의 영향력
Bernstein 다항식의 중요 성질:
1. b₀+b₁+b₂+b₃ = 1 (항상 합이 1 → 아핀 불변성)
2. 모든 bₖ ≥ 0 (음수 없음)
3. 따라서 p(u)는 항상 제어점들의 볼록 껍질(Convex Hull) 내부에 있음!

직접 계산 예제

제어점: p₀=(0,0), p₁=(1,2), p₂=(3,2), p₃=(4,0), u=0.5일 때 점 계산

Step 1: Bernstein 값 계산 (u=0.5)
b₀(0.5) = (0.5)³ = 0.125 × ... = (1-0.5)³ = 0.125
b₁(0.5) = 3×0.5×(0.5)² = 3×0.5×0.25 = 0.375
b₂(0.5) = 3×(0.5)²×0.5 = 0.375
b₃(0.5) = (0.5)³ = 0.125
Step 2: 합이 1인지 확인
0.125 + 0.375 + 0.375 + 0.125 = 1.0 ✓
Step 3: 점 계산
x = 0.125×0 + 0.375×1 + 0.375×3 + 0.125×4 = 0 + 0.375 + 1.125 + 0.5 = 2.0
y = 0.125×0 + 0.375×2 + 0.375×2 + 0.125×0 = 0 + 0.75 + 0.75 + 0 = 1.5
→ u=0.5일 때 점: (2.0, 1.5)

연속성 (Continuity)

연속성조건의미
C⁰끝점 위치 일치연결되어 있지만 꺾일 수 있음
C¹끝점 위치 + 미분 일치매끄럽게 연결
C²위치 + 1차 + 2차 미분 일치더 부드럽게 연결
G⁰= C⁰기하적 연속
G¹접선 방향 일치 (크기 무관)방향만 같으면 됨
⚠️ Bezier는 기본적으로 C⁰만 보장. C¹을 위해선 연결점에서 접선 조건을 수동으로 맞춰야 합니다.

B-Spline

💡 목표: Hermite의 매끄러움 + Bezier의 편리한 제어점 사용 + C² 연속성 자동 보장!

B-Spline의 핵심 개념

4개의 제어점 {pᵢ₋₂, pᵢ₋₁, pᵢ, pᵢ₊₁}으로 pᵢ₋₁과 pᵢ 사이의 곡선을 그립니다. 단, 곡선이 제어점을 통과하지 않습니다!

경계 조건 설정

자연스러운 연결을 위해 다음 조건을 사용합니다:

연속성: q(1) = p(0) = (1/6)(pᵢ₋₁ + 4pᵢ + pᵢ₊₁) 매끄러움: q'(1) = p'(0) = (1/2)(pᵢ₊₁ - pᵢ₋₁) 다음 구간: p(1) = (1/6)(pᵢ + 4pᵢ₊₁ + pᵢ₊₂) p'(1) = (1/2)(pᵢ₊₂ - pᵢ)
🧮 가중 평균: p(0) = (pᵢ₋₁ + 4pᵢ + pᵢ₊₁)/6 은 마치 주변 점들을 "1:4:1" 비율로 섞은 것. 자연스럽게 중간 어딘가에 곡선이 놓입니다.

B-Spline 기하행렬

c = Mₛ·p (여기서 p = {p₀, p₁, p₂, p₃}) Mₛ = (1/6)· ⎡ 1 4 1 0⎤ ⎢-3 0 3 0⎥ ⎢ 3 -6 3 0⎥ ⎣-1 3 -3 1⎦ p(u) = uᵀ·Mₛ·p

B-Spline 블렌딩 함수

b₀(u) = (1-u)³ / 6 b₁(u) = (4 - 6u² + 3u³) / 6 b₂(u) = (1 + 3u + 3u² - 3u³) / 6 b₃(u) = u³ / 6

세그먼트 추가

점 하나를 추가할 때마다 새 세그먼트가 생깁니다:

점 추가: pᵢ₊₂ 추가 새 세그먼트: {pᵢ₋₁, pᵢ, pᵢ₊₁, pᵢ₊₂} 사용 → pᵢ~pᵢ₊₁ 사이 곡선
⚠️ B-Spline은 각 세그먼트가 점 2개 사이만 커버. Bezier보다 세그먼트당 계산량이 많습니다.

곡선 방법론 비교 요약

방법 입력 통과 여부 연속성 특징
보간 (Interpolation) 4개 데이터 점 ✅ 모두 통과 C⁰ (기본) 연결점 꺾임 가능
Hermite 2 끝점 + 2 접선벡터 ✅ 끝점 통과 C¹ 가능 접선벡터 직접 지정
Bezier 4 제어점 ✅ 끝점만 통과 C⁰ 기본 직관적 조형, 볼록 껍질
B-Spline 4 제어점 ❌ 근사만 C² 자동! 가장 매끄러움, 로컬 제어

언제 어떤 방법을 쓰나?

상황추천이유
정확히 점을 통과해야 함보간 또는 Hermite제어점을 곡선이 지남
직관적인 모양 조형Bezier제어점이 곡선을 "당긴다"
매우 부드러운 곡선 필요B-SplineC² 연속성 자동 보장
접선 방향을 직접 알고 있음Hermite접선 정보를 직접 활용

행렬 관계

모든 방법: p(u) = uᵀ·M·(제어점) - 보간: M = Mᵢ = A⁻¹ - Hermite: M = Mₕ - Bezier: M = Mʙ - B-Spline: M = Mₛ
🔄 변환 가능: q = Mʙ⁻¹·M·p 공식으로 다른 방법의 제어점을 Bezier 제어점으로 변환할 수 있습니다!
